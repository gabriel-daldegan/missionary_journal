# Story 1.12: Standardize AIOX Pint Commands

## Status

Ready for Merge

```yaml
  - "npm run validate:structure"

  - "git diff --check"
  - "coderabbit review --agent --type uncommitted"
```

## Story

**As a** workflow maintainer,
**I want** AIOX handoffs, checklists, and story guidance to use a canonical Pint command convention,
**so that** CodeRabbit and agents stop producing recurring false-positive review churn around Sail Pint syntax.

## Acceptance Criteria

1. The story implementation documents one canonical repository-level formatting gate: `npm run lint`.
2. The story implementation documents that direct Pint invocation is only for focused formatter use, and the preferred direct Sail/Pint form is `vendor/bin/sail pint --dirty --format=agent`.
3. The story implementation distinguishes valid historical evidence from reusable future guidance: historical consumed stories and handoffs may remain unchanged, but current reusable guidance must not keep recommending non-canonical direct Pint forms.
4. AIOX-managed and project-maintained guidance that instructs future agents how to run Pint is updated consistently, including `AGENTS.md`, `docs/framework/coding-standards.md`, and any reusable AIOX checklist/template/workflow artifact found during inventory.
5. The implementation does not change Laravel application behavior, memory product behavior, routes, database schema, package dependencies, or user-facing UI.
6. Validation proves `npm run lint` still executes successfully through the package script and `npm run validate:structure` passes after guidance changes.
7. Validation includes a targeted search showing no current reusable guidance still recommends `vendor/bin/sail bin pint --dirty --format agent`, `vendor/bin/sail bin/pint`, or host-level `vendor/bin/pint` as the preferred command unless the occurrence is explicitly historical or intentionally annotated.

## Integration Verification

1. **IV1:** The existing `package.json` `lint` script remains the source of truth for the repo-level formatting gate and still runs successfully.
2. **IV2:** AIOX structure validation passes after updating story/handoff/checklist/template guidance.
3. **IV3:** The final diff is limited to workflow/documentation guidance and story/handoff artifacts; no PHP application code, migrations, routes, Livewire components, Filament pages, Blade UI, tests for product behavior, or package dependency changes are included.
4. **IV4:** CodeRabbit review does not reopen the same unresolved Pint command false positive against newly edited reusable guidance.

## Implementation Interpretation

Story 1.12 is backlog-driven technical debt, not a new Memory MVP product feature. Epic 1 product scope is complete through Story 1.11; this story cleans up recurring AIOX command guidance that caused CodeRabbit review churn during Story 1.7 and later publication flows. [Source: docs/stories/backlog.json#1781200916741] [Source: .aiox/handoffs/20260623T220947-0300-devops-to-sm-story-1-12-ready-to-draft.yaml#next-story]

The canonical policy for future AIOX handoffs and checklists is:

- Use `npm run lint` as the repo-level formatting/quality gate because it is the package script already maintained for this repository.
- Use `vendor/bin/sail pint --dirty --format=agent` only when direct focused Pint execution is required.
- Treat `vendor/bin/sail bin pint --dirty --format agent` as a historically verified valid form in this repo, but not the preferred reusable guidance going forward.
- Treat host-level `vendor/bin/pint` and `vendor/bin/sail bin/pint` as non-canonical for future AIOX guidance unless an implementation note explicitly explains a narrow reason.

This story should reconcile wording, not break working commands. Prior CodeRabbit findings were false-positive or non-blocking because `vendor/bin/sail bin pint --dirty --format agent` executed successfully, but keeping multiple forms in future handoffs continues to create review noise. [Source: docs/stories/1.7.trip-period-record-creation.story.md#qa-results] [Source: docs/stories/1.10.english-portuguese-and-spanish-ui-localization.story.md#dev-agent-record] [Source: .aiox/handoffs/20260623T220947-0300-devops-to-sm-story-1-12-ready-to-draft.yaml#validation-context]
