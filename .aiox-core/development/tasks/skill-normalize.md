# Skill Normalize

**Task ID:** skill-normalize
**Agent:** skill-standardizer
**Purpose:** Convert non-compliant Codex/OpenAI skills into concise, progressive-disclosure skill packages.

## Inputs

- `target`: a skill directory, a `SKILL.md` path, or `all`
- `mode`: `plan` or `write` (default: `plan`)
- `scope`: `local`, `global`, or `all` when target is `all`

## Procedure

1. Run `skill-audit` for the target scope.
2. For each flagged skill, read the current `SKILL.md` and list the minimum required changes.
3. Normalize frontmatter:
   - `name` must be kebab-case.
   - `description` must be third-person, specific, trigger-rich, and <= 1024 characters.
   - Remove XML tags and long examples from `description`.
4. Normalize body:
   - Keep a quick start and workflow.
   - Keep `SKILL.md` under 500 lines.
   - Move long manuals, examples, and reference data to `references/`.
   - Move deterministic repeatable logic to `scripts/`.
   - Move reusable output assets to `assets/`.
5. Preserve user material:
   - Do not delete content. Move it into bundled resources when it is still useful.
   - Keep filenames stable unless a rename is required for kebab-case compliance.
6. Write behavior:
   - In `plan` mode, show a patch plan only.
   - In `write` mode, edit local repo files directly.
   - For `~/.codex/skills`, ask explicit approval before editing because it is global state.

## Output

Report:
- files changed or planned
- frontmatter before/after summary
- content moved to references/scripts/assets
- remaining risks or manual review notes

## Veto Conditions

Stop and ask before writing when:
- the target is under `~/.codex/skills`
- the skill appears generated by a third-party package and no local override exists
- normalization would remove domain-specific instructions rather than move them
- the user's requested scope is ambiguous
